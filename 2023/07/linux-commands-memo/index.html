Linux Commands Memo

CuSO4_Deposit
2023-07-16 (Updated: 2023-07-17)
Linux

arch

print the computer architecture.

e.g.

cat

concatenate files and print on the standard output.

cat [OPTION]… [FILE]…`

 -A, --show-all equivalent to -vET.

-b, --number-nonblank number nonempty output lines, overrides -n.

-E, --show-ends display $ at end of each line.

-n, --number number all output lines.

-T, --show-tabs display TAB characters as ^I.

-v, --show-nonprinting use ^ and M- notation, except for LFD and TAB.

chmod

change file mode bits.

chmod [OPTION]… MODE[,MODE]… FILE…

chmod [OPTION]… OCTAL-MODE FILE…

chmod [OPTION]… –reference=RFILE FILE…

chown

change file owner and group.

chown [OPTION]… [OWNER][:[GROUP]] FILE…

chown [OPTION]… –reference=RFILE FILE…

chsh

change login shell.

-s, --shell <shell> specify the user’s new login shell. The command must be in /etc/shells

files

   /etc/passwd
       User account information.

   /etc/shells
       List of valid login shells.

   /etc/login.defs
       Shadow password suite configuration.

curl

https://catonmat.net/cookbooks/curl

-A, --user-agent <name> specify the user-agent.

-d, --data <date> send a POST request.

e.g.

 -f, --fail fail silently (no output at all) on server errors. This is mostly done to enable scripts etc to better deal with failed attempts.

-F, --form <name=content> send a form.

e.g.

-G, --get send a GET request. This will make all -d options act as GET params.

-H, --header <header/@file> send header.

e.g.

-I, -head fetch the headers only.

-K, --config <file> use a curl config file.

-L, --location follow redirections. If the server reports the data has moved (http status code 3xx), curl redos the request to the new place.

-o, --output <file> write output to the file instead of stdout.

-O, --remote-name write output to a local file named like the remote file we get.

-s, --silent quiet mode, do not display progress meter or error messages.

-S, --show-error used with -s, let curl still shows error messages.

-u, --user <user:password> specify credentials for authentication. If only username is specified, curl will prompt for the password.

--url specify a URL, useful when using curl config.

-X, --request <method> specify the method.

e.g.

-4, --ipv4 use ipv4 only. same for -6, --ipv6.

-#, --progress-bar Make curl display transfer progress as a simple progress bar instead of the standard, more informational, meter.

tar

an archiving utility.

tar -c [-f ARCHIVE] [OPTIONS] [FILE…]

-A, --catenate, --concatenate append archive to the end of another archive.

-c, --create create a new archive, directories are archived recursively if no-recursion is not given.

-f, --file=<Archive> use archive file.

-t, --list list the contents of an archive.

-x, --extract, --get extract files from an archive.

e.g.

touch

change file timestamps.

touch [OPTION]… FILE…

A FILE argument that does not exist is created empt~~~~y, unless -c or -h is supplied.

-a change only the access time.

-c, --no-create do not create any files.

-d, --date=STRING parse STRING and use it instead of current time.

-m change only the modification time.

-t STAMP use [[CC]YY]MMDDhhmm[.ss] instead of current time.

unzip

list, test and extract compressed files in a ZIP archive.

unzip [-Z] [-cflptTuvz[abjnoqsCDKLMUVWX$/:^]] file[.zip] [file(s) …] [-x xfile(s) …] [-d exdir]

-d specify extract directory.

-l list archive files (short format).

-o overwrite existing files without prompting. This is a danferous option.

-v list archive files (verbose format)

zip

package and compress (archive) files.

zip [-aABcdDeEfFghjklLmoqrRSTuvVwXyz!@$] [–longoption …] [-b path] [-n suffixes] [-t date] [-tt date] [zipfile [file …]] [-xi list]

-r, --recursive-paths recursively archive a dir.
